# Galley Obsidian 插件设计

- 状态：已确认，待实施计划
- 日期：2026-07-14
- 产品名：Galley
- 许可证：AGPL-3.0

## 1. 摘要

Galley 是一个 AI 驱动的 Obsidian 文章排版插件。用户对当前 Markdown 执行一次命令后，插件让 OpenAI 兼容模型加载 `gzh-design-skill`，由模型直接生成主题化 HTML，再在 Obsidian 中完成所见即所得编辑、独立保存和多配置导出。

首版内置原 Skill 的六套主题，支持 AI 生成自定义主题。生成后的 HTML 与原 Markdown 独立；后续编辑不会回写 Markdown。桌面端提供完整生成和可视化编辑，移动端仅提供安全预览。

Galley 不把产品限定为微信公众号排版。它保留原 Skill 的主题、组件、章节、关键词和质量方法，并增加 Galley Authoring、标准网页、便携内联和微信公众号兼容四种文档契约。其中 Authoring 是主编辑格式，后三者是导出格式。

## 2. 背景与约束

### 2.1 参考项目

- [`isjiamu/gzh-design-skill`](https://github.com/isjiamu/gzh-design-skill) 提供 Skill 工作流、六套主题组件库、主题生成器和确定性校验规则。本设计基于本地检出的提交 `ba1f4175519b481cb3566616c9e5178705067904`。
- [`mandolin/obsidian-html-v-editor`](https://github.com/mandolin/obsidian-html-v-editor) 证明了 HugeRTE、本地资源打包、安全预览和 Obsidian HTML 自定义视图的可行性。本设计参考的提交为 `cefca3c613c8695a55b6ada016639b63c9f303aa`。

### 2.2 许可

`gzh-design-skill` 使用 AGPL-3.0。Galley 会直接复用和改造其 Skill、主题和校验资产，因此整个插件使用 AGPL-3.0，保留原作者版权和联名署名，并在发行包中提供完整许可证和源代码地址。

`obsidian-html-v-editor` 使用 MIT。Galley 不依赖用户安装该插件；如实施时复用其代码，将保留对应 MIT 版权声明。HugeRTE、DOMPurify 等依赖的许可证进入第三方声明清单。

### 2.3 Obsidian 兼容范围

- `minAppVersion`：1.11.4。该版本提供本设计依赖的 `App.secretStorage`。
- `isDesktopOnly`：`false`，使移动端能够加载只读预览能力。
- 桌面端：生成、主题实验室、可视化编辑、源码高级模式、导出。
- 移动端：仅安全预览，不初始化 AI 生成、HugeRTE 编辑或 Skill 导入能力。

## 3. 目标

1. 当前 Markdown 一键生成高质量主题化 HTML。
2. 每次生成都让模型实际加载并遵循 `gzh-design-skill`，而不是只使用一段简化 Prompt。
3. 支持任意实现 OpenAI Chat Completions 兼容接口的服务，并优先使用工具调用加载 Skill。
4. 用户无需编辑 HTML 代码即可修改内容和常用文章组件。
5. 同一份主 HTML 可导出标准网页、便携内联和微信公众号兼容结果。
6. 内置六套主题，并支持以文字描述或参考图生成、校验和保存自定义主题。
7. 所有模型结果都经过安全清理、内容完整性检查和质量校验。
8. 生成、编辑、失败恢复和导出均不破坏原 Markdown 或既有 HTML。

## 4. 非目标

首版不包含：

- HTML 与 Markdown 双向同步；
- 重新生成后自动重放人工编辑补丁；
- 移动端 AI 生成或可视化编辑；
- Word、PDF、纯文本等非 Markdown 输入归一化；
- 交互式网页脚本、表单应用或落地页生成；
- 实时多人协作；
- 后台自动更新 Skill；
- 依赖另一个 Obsidian 插件才能完成核心工作流。

## 5. 核心决策

| 主题 | 决策 |

| AI 方式 | AI 驱动，模型直接生成 HTML |

| 输出 | Galley Authoring 主文件，加三种导出配置 |

| 工作区 | 一体化工作台：左流程、中画布、右属性 |

## 6. 系统架构

Galley 分为以下模块。模块只通过明确接口协作，避免 UI、模型调用、文件格式和平台转换互相渗透。

### 6.1 Obsidian 集成层

职责：

- 注册命令、Ribbon 入口、文件菜单和自定义 ItemView；
- 识别当前 Markdown 或 Galley HTML；
- 按平台门控桌面和移动能力；
- 将 vault 文件事件转交给会话和冲突管理模块。

主要命令：

- `Galley: AI 排版当前文章`
- `Galley: 在工作台中打开`
- `Galley: 打开主题实验室`
- `Galley: 导出当前文章`
- `Galley: 导入 Skill 包`
- `Galley: 检查模型连接与能力`

### 6.2 Skill Runtime

职责：

- 管理内置和用户导入的 Skill 包；
- 提供只读虚拟文件系统；
- 暴露 `read_skill_file(path)` 模型工具；
- 强制记录 Skill 版本和实际加载文件；
- 在工具不可用或模型不遵循工具协议时完成上下文注入回退；
- 把内置主题索引与用户主题覆盖层合并为模型可见索引。

### 6.3 OpenAI 兼容网关

职责：

- 管理 Base URL、Secret 名称、模型名、温度、超时和上下文窗口配置；
- 实现 Chat Completions 消息、SSE 流式响应和工具调用协议；
- 探测工具、流式和视觉能力；
- 统一取消、重试、错误映射和敏感信息脱敏；
- 流式不可用时自动使用非流式响应。

### 6.4 AI 生成管线

职责：

- 解析 Markdown 并注入临时来源标识；
- 建立 Skill 会话、完成主题决策和直接 HTML 生成；
- 提取模型响应中的 HTML；
- 调用安全清理、完整性检查、主题检查和自动修复；
- 对超出上下文窗口的文章按章节批次生成 HTML 片段，再完成全篇装配和校验。

### 6.5 主题仓库

职责：

- 提供六套内置主题和通用组件；
- 管理用户自定义主题的元数据、组件库和预览；
- 生成合并主题索引；
- 运行组件库检查；
- 支持主题导入、导出和禁用。

### 6.6 HTML 工作区

职责：

- 管理 Authoring 文档的打开、解析、可视化编辑和保存；
- 桌面端通过本地打包的 HugeRTE 提供所见即所得编辑；
- 保留完整文档外壳、内联样式和 `data-galley-*` 属性；
- 提供段落类型、组件切换、链接、图片、表格和主题属性操作；
- 提供预览和可选的高级源码模式；
- 处理自动保存、外部修改冲突和本地版本历史。

### 6.7 导出管线

职责：

- 从当前 Authoring 文档派生三种输出；
- 执行确定性标签、属性、样式和文档外壳转换；
- 按配置运行校验；
- 微信公众号配置仍失败时，调用已加载 Skill 的模型修复导出副本；
- 保存文件或把渲染后的富文本复制到剪贴板。

## 7. 文件与数据模型

### 7.1 插件内置资产

内置 Skill 固定在发行包中，保持原目录结构和相对路径。插件不在运行时修改该目录。

```text
assets/
  skills/
    gzh-design/<pinned-version>/
      SKILL.md
      references/
      scripts/
      assets/
  profiles/
    galley-authoring.md
    standard-web.md
    portable-inline.md
    wechat.md
```

用户手动导入的 Skill 包保存在插件自己的数据目录，不写入普通文章目录。导入要求：

- 存在 `SKILL.md`、`references/theme-index.md`、`references/common-components.md`；
- 路径不能越出包根目录；
- 拒绝符号链接和 ZIP 路径穿越；
- 解压后总大小不得超过 25 MiB；
- 能够完成主题索引解析和组件自检；
- 用户明确选择后才切换活动版本。

### 7.2 文章产物

默认与来源 Markdown 保存在同一目录。用户可在设置中指定统一输出目录。

```text
文章名.galley.html
文章名.galley.json
文章名.web.html
文章名.inline.html
文章名.wechat.html
```

`文章名.galley.html` 是完整 HTML 文档，包含 `DOCTYPE`、语言、head 元数据和 body。文章样式以行内样式为主，不允许脚本。`data-galley-*` 属性保存来源段落、组件角色和编辑器需要的语义。

HugeRTE 只编辑 body 内容；文档编解码器在保存时重新组合未经编辑器管理的文档外壳。

`文章名.galley.json` 至少包含：

```json
{
  "schemaVersion": 1,
  "documentId": "uuid",
  "sourcePath": "文章名.md",
  "sourceHash": "sha256",
  "htmlHash": "sha256",
  "themeId": "graphite-minimal",
  "skillVersion": "git-sha-or-import-hash",
  "skillLoadMode": "tool-calls",
  "skillFiles": [],
  "model": "provider-model-id",
  "promptVersion": 1,
  "generatedAt": "ISO-8601",
  "validation": {},
  "exports": []
}
```

元数据不得包含 API Key、Authorization Header、完整请求日志或用户未选择发送的本地文件内容。

### 7.3 自定义主题

```text
.galley/
  themes/
    <theme-id>/
      theme.json
      component-library.md
      preview.html
  history/
    <document-id>/
      <timestamp>.html
```

自定义主题存入 vault，便于同步和备份。内置 Skill 的 `theme-index.md` 保持只读；运行时生成合并索引。版本历史默认保留每篇文章最近 20 个保存版本，超出后删除最旧版本。

主题 ID 只能使用小写字母、数字和连字符，且不得与内置或既有主题冲突。Skill Runtime 会把 `component-library.md` 只读挂载为虚拟路径 `references/theme-<theme-id>.md`，合并索引也指向该虚拟路径。因此模型仍按原 Skill 的标准路径读取主题，不需要知道 vault 的真实位置。`theme.json` 记录 AGPL-3.0 和原项目署名。

## 8. Skill 加载协议

### 8.1 工具调用优先

每个生成任务创建独立会话。系统消息声明活动 Skill、Galley 输出配置和工具协议。模型必须首先调用：

```text
read_skill_file("SKILL.md")
```

随后按工作流读取：

- `references/theme-index.md`；
- 选定主题的 `references/theme-*.md`；
- `references/common-components.md`；
- 自定义主题时的 `references/theme-generator.md`；
- Skill 明确引用且本任务需要的其他文件。

工具实现仅接受规范化的相对路径。`..`、绝对路径、URL、符号链接和未登记文件均被拒绝。活动 Skill 文件和已登记用户主题的只读虚拟挂载构成完整白名单。每次读取进入会话审计记录。

如果模型声称已加载却未请求必需文件，插件再次要求完成读取；第二次仍未完成则切换到上下文注入回退。

### 8.2 上下文注入回退

以下情况触发回退：

- 模型或端点不支持工具调用；
- 工具调用响应格式不兼容；
- 模型两次未按协议读取必需文件；
- 能力测试已将该模型标记为工具不可用。

回退上下文完整包含本任务必需的 `SKILL.md`、主题索引、选定主题库、通用组件库、输出配置和其他直接依赖文件。它不会注入与当前任务无关的其余五套主题，以控制上下文长度。

### 8.3 指令优先级

1. 用户对当前文章的明确要求；
2. Galley 当前输出配置中与平台有关的补充协议；
3. `gzh-design-skill` 的工作流和主题组件规则。

Galley 配置只能覆盖平台外壳、允许标签和导出约束，不得覆盖原 Skill 的主题选择、组件使用、章节编号、关键词标记、内容保真和质量校验方法。

## 9. AI 生成流程

### 9.1 预检

1. 确认当前文件为 Markdown 且存在有效内容。
2. 读取模型配置和 SecretStorage 中的密钥。
3. 估算 Markdown、Skill 必需文件和响应的上下文用量。
4. 为标题、段落、列表、代码块、表格和图片分配临时来源标识，如 `h001`、`p002`、`img003`。
5. 解析 Obsidian wiki 链接和本地资源；普通文章生成只发送 vault 相对路径、alt 和尺寸信息，不发送绝对系统路径或图片二进制。

### 9.2 主题决策

在全自动命令下，模型读取 Skill 和主题索引后返回主题 ID、文章类型和简短理由。用户已在工作区手动指定主题时跳过决策调用。

主题决定后，模型必须读取该主题组件库和通用组件库，才允许进入生成步骤。

### 9.3 直接生成 HTML

模型接收完整 Markdown、Galley Authoring 配置和来源标识契约，直接返回完整 HTML 文档，不返回 Markdown 代码围栏或解释文字。

模型响应可以流式传输，但未闭合 HTML 不进入执行环境。工作区显示阶段、已接收 token 和取消按钮；只有完整响应经过清理后才渲染。

### 9.4 上下文超限

用户设置模型上下文窗口，默认值为 128,000 tokens。预检预计超过 85% 时进入长文模式：

1. 保持同一主题、Skill 版本和 Authoring 配置；
2. 按二级标题边界拆分文章；
3. 每批由模型直接生成 HTML 片段；
4. 插件按来源顺序组合完整文档；
5. 模型完成一次全篇一致性检查；
6. 插件运行全部确定性校验。

### 9.5 校验与修复

依次执行：

1. HTML 可解析性和文档外壳检查；
2. DOM 安全清理；
3. 来源标识完整、唯一和顺序检查；
4. 图片、链接和资源引用检查；
5. 主题组件、内联样式和结构规则检查；
6. 章节编号、关键词标记、目录、签名区和内容保真检查；
7. 当前输出配置的规则检查。

失败时把原 HTML、精确错误列表和缺失来源片段发送给同一 Skill 会话。最多自动修复两轮。两轮后仍失败则保存为“未验证草稿”，同时写入诊断报告，不覆盖已有通过验证的文档。

### 9.6 落盘

校验成功后先写临时文件，再通过 vault rename 完成替换。主 HTML 和 sidecar 均成功后才更新工作区状态。重新从 Markdown 生成时默认使用带序号的新文件名；只有用户明确确认才覆盖现有主 HTML。

## 10. 工作区与交互

### 10.1 一体化布局

用户已确认方案 A：

- 顶部：文件状态、主题选择、重新生成、预览和导出；
- 左侧：生成、编辑、导出流程，文章结构和版本历史；
- 中间：模拟实际阅读宽度的文章画布；
- 右侧：内容、页面和导出属性。

默认命令不弹配置对话框。它按以下路径运行：

```text
AI 排版当前文章
→ 模型加载 Skill
→ AI 自动选主题
→ 直接生成 HTML
→ 校验并自动修复
→ 打开可视化编辑
```

高级模型配置位于插件设置，主题和文章参数位于右侧面板。

### 10.2 可视化编辑

桌面端使用本地打包 HugeRTE。首版提供：

- 直接修改标题、正文、列表和引用；
- 加粗、斜体、下划线、链接和文字颜色；
- 图片替换、alt、说明和对齐；
- 表格行列操作；
- 在正文、引言、金句、提示卡等已注册组件之间切换；
- 页面宽度、段落间距和主题允许的样式属性；
- 撤销、重做、保存和版本恢复。

选择文本时出现轻量浮动工具栏。右侧面板负责块级组件和页面属性。用户无需打开源码模式完成上述操作；源码模式只作为高级和故障排查入口。

### 10.3 保存与冲突

- 编辑后 800 ms 防抖自动保存；
- 显式保存命令立即写入；
- 每次覆盖前比较 sidecar 中的 `htmlHash`；
- 检测到外部修改时停止自动保存，提供“重新载入”“另存副本”“明确覆盖”；
- 成功覆盖前写入版本历史，默认保留 20 个版本。

### 10.4 移动端

移动端打开 `.galley.html` 时使用 DOMPurify 清理并在无脚本 sandbox iframe 中预览。生成、主题实验室、编辑、Skill 导入和导出修复命令不注册或显示不可用说明。

## 11. 自定义主题工作流

1. 用户打开主题实验室，输入必填风格描述和可选名称、颜色、字体等参数。
2. 用户可选一张参考图；只有模型能力测试确认视觉输入可用时显示该入口。
3. 模型加载 `SKILL.md`、`theme-generator.md`、主题索引和通用组件。
4. 模型按原 Skill 工作流生成完整组件预览和标准主题库文件。
5. 插件运行组件检查，展示包含全部区块的整页预览。
6. 用户可以重新生成或确认保存；未确认内容不写入主题仓库。
7. 保存后生成 `theme.json`、`component-library.md` 和 `preview.html`，并加入运行时合并索引。
8. 新主题与内置主题同权，可用于自动推荐、手动选择和后续生成。

## 12. 导出配置

### 12.1 Galley Authoring

主编辑文件使用完整 HTML 文档，允许文章需要的语义标签、链接、表格和媒体占位；保留内联样式和 `data-galley-*`。禁止脚本、事件属性、危险 URL、表单执行和任意 iframe。

### 12.2 标准网页

输出 `文章名.web.html`：

- 完整 `DOCTYPE/html/head/body`；
- 标题、语言、字符集和 viewport；
- 响应式阅读容器；
- 保留语义标签、链接、表格和安全媒体；
- 不允许脚本。

### 12.3 便携内联 HTML

输出 `文章名.inline.html`：

- 无文档外壳的正文片段；
- 样式全部内联；
- 无外部 CSS、字体或脚本依赖；
- 移除 `data-galley-*` 编辑元数据；
- 支持复制渲染后的富文本到普通 CMS、邮件或其他富文本编辑器。

### 12.4 微信公众号兼容

输出 `文章名.wechat.html`：

- 恢复原 Skill 的平台红线；
- 输出纯 `<section>` 正文片段；
- 样式全部内联；
- 文字节点使用 `<span leaf="">`；
- 运行移植后的 `validate_gzh_html` 等价校验；
- 确定性转换失败时，由已加载 Skill 的模型修复导出副本，最多两轮；
- 不修改 Authoring 主文件。

### 12.5 扩展性

三种导出实现统一的 `ExportProfile` 接口，包含转换、验证、命名和复制策略。首版不提供用户自定义规则编辑器，但架构允许后续增加新平台配置。

## 13. 安全设计

### 13.1 密钥

使用 Obsidian `SecretStorage` 和 `SecretComponent`。插件设置只保存 Secret ID，调用时通过 `app.secretStorage.getSecret()` 获取。API Key 不进入普通日志、sidecar、错误报告或测试快照。

### 13.2 网络

- Base URL 由用户配置；
- HTTPS 是远程端点的默认要求；
- HTTP 仅允许 `localhost`、回环地址和 RFC1918 局域网地址，并显示风险提示；
- 请求日志对 Authorization、API Key、Cookie 和用户正文脱敏；
- 普通文章生成不上传本地图片二进制；
- 主题参考图必须由用户明确选择后才上传。

### 13.3 模型 HTML

所有模型返回均视为不可信：

- 在任何预览或编辑器加载前解析和清理；
- 移除 script、事件属性、javascript URL、object、embed、执行型 iframe 和危险表单；
- 限制 data URL 类型和大小；
- 外部资源按设置和安全策略处理；
- 预览使用 sandbox iframe；
- 编辑器只接收清理后的 body HTML。

### 13.4 Skill 和 ZIP

- 工具只能读取活动 Skill 根目录内登记文件；
- 导入包拒绝路径穿越、符号链接、超限大小和缺失必需文件；
- Skill 脚本不会作为任意 Python 或 shell 代码执行；校验逻辑移植为插件内受控 TypeScript 实现；
- 自定义主题只有用户确认后才写入 vault。

## 14. 错误处理与恢复

| 场景 | 行为 |
| --- | --- |
| 401/403 | 停止，提示密钥或权限问题，不重试 |
| 429/5xx/网络中断 | 指数退避，最多重试两次 |
| 用户取消 | 终止请求，保留原文件，不把部分 HTML 当成结果 |
| 工具调用不可用 | 切换必需 Skill 文件完整注入 |
| Skill 缺失或损坏 | 拒绝生成，列出缺失或无效文件 |
| 上下文过长 | 按章节批次直接生成 HTML，再全篇校验 |
| HTML 畸形或漏段 | 同一 Skill 会话自动修复两轮 |
| 修复仍失败 | 保留未验证草稿和诊断，不覆盖已验证版本 |
| 外部文件修改 | 暂停自动保存，要求重载、另存或明确覆盖 |
| 自定义主题校验失败 | 保留预览草稿，不登记主题 |
| 导出校验失败 | 保留主文件，报告失败的导出副本 |

## 15. 测试策略

### 15.1 单元测试

覆盖：

- Skill 包解析、路径规范化、白名单和合并主题索引；
- 工具调用状态机和上下文注入回退；
- Prompt 分层与来源标识注入；
- 模型 HTML 提取、安全清理和文档编解码；
- 来源完整性、唯一性和顺序检查；
- 三种导出转换和验证；
- Obsidian 资源路径解析；
- sidecar schema、哈希、冲突和版本保留；
- 密钥及日志脱敏。

### 15.2 模型协议测试

使用本地模拟 OpenAI 兼容服务覆盖：

- 流式与非流式响应；
- 正常工具调用、多轮工具调用和无工具端点；
- 模型忽略必需 Skill 读取；
- 视觉能力存在和缺失；
- 401、403、429、500、超时、中断和取消；
- 代码围栏、解释文字、空响应和畸形 HTML。

### 15.3 黄金样例

使用原 Skill 示例文章、六套主题和录制的模型响应，检查标题、段落、图片、代码、列表、表格、引用、章节编号、关键词、目录和签名区。对主题组件和最终 HTML 运行移植后的等价校验。

模型输出具有随机性，因此默认 CI 不比较实时模型的完整 HTML 快照。真实端点烟雾测试通过环境变量显式启用，不使用开发者个人密钥，也不进入默认 CI。

### 15.4 Obsidian 集成测试

使用测试 vault 覆盖：

- 命令和视图注册；
- Markdown 到文章文件和 sidecar 的落盘；
- HugeRTE 加载、修改、保存和恢复；
- 外部修改冲突；
- 自定义主题持久化；
- 三种导出；
- 桌面完整功能与移动只读门控。

### 15.5 长文基准

至少使用一篇约 10,000 个中文字符、包含本地图片、代码块、表格和十个以上二级标题的文章。基准要求内容来源标识零遗漏、取消和失败不破坏文件，且长文模式能够完成全篇验证。

## 16. 首版验收标准

1. 对当前 Markdown 执行一次命令后，模型加载活动 Skill，并在 sidecar 中记录加载模式、版本和文件清单。
2. 六套内置主题均能生成可解析且内容完整的 Authoring HTML。
3. 工具调用不可用或未被模型正确使用时，必需 Skill 文件完整注入回退能完成同一流程。
4. 用户无需接触源码即可修改文字、标题、列表、链接、图片、表格和已注册主题组件，并可靠保存。
5. 同一主文件可导出标准网页、便携内联和微信公众号兼容结果。
6. 自定义主题可由 AI 生成、整页预览、校验、保存和复用。
7. 移动端只能安全预览，不能发起生成或进入可视化编辑。
8. API Key 不出现在插件配置文件、日志、生成文件、诊断或测试产物中。
9. 取消、网络失败、校验失败和外部冲突不会破坏原 Markdown 或已有 HTML。
10. 发布包包含 AGPL-3.0、原项目署名、第三方许可证、`main.js`、`manifest.json` 和 `styles.css`。

## 17. 实施里程碑

该产品由相互依赖的多个子系统组成，实施按可验证的纵向切片推进：

1. **基础与 Skill Runtime**：插件骨架、设置、SecretStorage、OpenAI 兼容网关、Skill 包读取、工具调用和注入回退，以及模拟模型服务。
2. **生成闭环**：来源标识、主题决策、六套内置主题、直接 HTML 生成、清理、校验、自动修复、sidecar 和安全落盘。完成后可从命令生成并打开只读结果。
3. **一体化工作台**：完整布局、HugeRTE、Authoring 文档编解码、自动保存、冲突和版本历史。完成后达到桌面端可视化编辑目标。
4. **多配置交付**：标准网页、便携内联、微信公众号兼容、复制富文本和移动端安全预览。
5. **主题实验室与发布**：自定义主题生成、预览、虚拟挂载、Skill 导入、长文基准、许可证清单和发布门禁。

每个里程碑必须通过对应单元、协议和集成测试后再进入下一阶段；实现计划可以拆成多个提交，但不改变本设计中的统一文件格式和模块接口。
